# Test Task — 3D Prototype on Unity

Небольшой 3D-прототип, реализующий базовую архитектуру(R3, DI, UniTask) и базовые игровые механики: передвижение игрока, автоматическую атаку ближайшего противника в зоне видимости, патрулирование врагов, респавн и UI-счетчик убийств.

## Что реализовано

### Игрок
- Передвижение на `WASD`
- Камера от первого лица
- Автоматическая атака при остановке
- Поиск цели выполняется в радиусе атаки через `OverlapSphere`
- Из найденных целей выбирается **ближайший живой противник**, попавший в сектор взгляда камеры

### Противники
- Спавнятся в количестве, заданном в `EnemyConfig`
- Патрулируют по заранее заданным маршрутам
- Для каждого врага:
  - маршрут выбирается из доступных на сцене
  - стартовая точка маршрута выбирается случайно
  - направление обхода маршрута также выбирается случайно

### UI
- Отображается счетчик убитых противников

## Использованный стек
- **Unity**
- **C#**
- **UniTask** — для асинхронных игровых операций вместо корутин
- **R3** — для реактивной логики на уровне событий и состояния
- **Custom DI Container** — для связывания зависимостей на сцене

## Структура логики

### Bootstrap
`GameplayBootstrap`
- собирает зависимости сцены
- создает и регистрирует сервисы
- связывает игрока, спавнер врагов и UI

### Игрок
`PlayerRoot`
- корневой объект игрока, связывает дочерние компоненты

`PlayerMovement`
- отвечает только за перемещение игрока

`PlayerAutoAttack`
- отвечает только за автоматическую атаку
- при остановке игрока запрашивает лучшую цель у сервиса поиска
- выполняет атаку с заданным интервалом

`TargetSelectionService`
- ищет цели через `Physics.OverlapSphere`
- отбирает только живых врагов
- проверяет попадание цели в сектор взгляда камеры
- возвращает ближайшую подходящую цель

### Враги
`EnemySpawner`
- создает врагов по данным из `EnemyConfig`
- назначает им маршруты

`EnemyRoute`
- хранит набор заранее заданных точек патрулирования

`EnemyPatrol`
- двигает врага по маршруту
- поддерживает случайный выбор направления обхода

`EnemyAgent`
- объединяет здоровье, патруль, смерть и респавн

`EnemyHealth`
- отвечает за HP
- публикует событие смерти через R3

### UI
`KillCounter`
- реактивная модель счетчика убийств

`KillCounterView`
- подписывается на `KillCounter` и обновляет UI

## Почему выбрана такая логика выбора цели

Так как проект ориентирован на вакансию, связанную с VR, автоатака сделана не просто по радиусу вокруг игрока, а с учетом **направления взгляда камеры**.

Алгоритм выбора цели:
1. В радиусе атаки собираются кандидаты через `OverlapSphere`
2. Из них остаются только живые враги
3. Проверяется, попадает ли враг в сектор взгляда камеры
4. Выбирается ближайший враг

Такое поведение выглядит более естественно для first-person / VR-подхода, чем атака врагов за спиной.
